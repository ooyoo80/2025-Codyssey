# Merge vs Rebase

## 🔁 Merge
- 두 브랜치의 변경 사항을 통합.
- 기존 커밋 히스토리는 유지되며, **merge commit**이 생성됨.
- 협업 시 흔히 사용되는 방식.

---

## 🔀 Rebase
- 브랜치를 다른 브랜치의 끝으로 이동시키고 히스토리 재작성.
- 커밋 로그가 더 깔끔해짐.
- 하지만 커밋 히스토리가 변경되어 공용 브랜치에는 위험.

---

## ✅ 협업 시 merge를 추천하는 이유

- 안정성: 커밋 히스토리를 유지하므로, 이력이 보존됨.
- 문제 추적 용이: 충돌 시점, 해결 과정이 명확함.
- 팀원 간 충돌 방지: rebase는 히스토리를 바꾸므로 push 시 충돌 발생 가능.
